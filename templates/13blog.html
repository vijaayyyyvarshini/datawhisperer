OOPS (Object Oriented Programming) is programming paradigm that uses objects, which are instances of classes, to design and structure software. OOPS is based on several key principles and concepts that help organize and manage code in a more modular and reusable way.

With OOPS, we have classes and objects.

My principles of software development class involved me to design a GAME with using these OOPS concept in the narratives.

I used:

Classes

A class is a blueprint or template for creating objects. It defines the properties (attributes) and behaviors (methods) that objects of that class will have.


Create_character is a class that I defined to create all my characters in the game.


Objects

An object is an instance of a class. It represents a real-world entity and encapsulates both data (attributes) and behavior (methods).


Here, Evander, lib, Castle, King, Merchant are all examples of objects initialised.

Encapsulation

Encapsulation is the practice of bundling data (attributes) and the methods (functions) that operate on that data into a single unit, known as a class. It hides the internal state of an object and exposes only the necessary functionality through methods.


In the above code, I have used Encapsulation, as I have bundled up all the methods - CreateCharacter, setClothing, setExpression, and setHairstyle into one class.

Inheritance

Inheritance allows one class (subclass or child class) to inherit the attributes and methods of another class (superclass or parent class). It promotes code reuse and establishes a hierarchy of classes.


In my above diagram, I have done Inheritance while defining the user stories. This helps me to make use of parent class properties and resuse the code.

Abstraction

Abstraction is the process of simplifying complex reality by modeling classes based on the essential properties and behaviors of objects. It involves ignoring the irrelevant details and focusing on what's important.


In the above code, I have defined two abstract classes and abstraction methods.

Constructors and Destructors

A constructor is a special method in a class that is called when an object is created. It initializes the object's attributes.

A destructor is a method that is called when an object is destroyed. In some programming languages, like C++, destructors are used for resource cleanup.

I didn't explicitly define any Constructors and Destructors in my code.
